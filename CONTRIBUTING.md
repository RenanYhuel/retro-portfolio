# Contributing

Merci d'envisager une contribution ! Voici quelques règles simples pour un repo propre.

## Signaler un bug

- Ouvre une issue et fournis : description claire, étapes pour reproduire, environnement (OS, Node version), logs si pertinent.

## Demandes de fonctionnalité

- Ouvre une issue avec la motivation et un petit wireframe ou exemple d'utilisation.

## Branches

- Utilise des branches descriptives : `feat/<short-description>`, `fix/<short-description>`, `chore/<short>`.
- Rebase ou merge régulièrement depuis `main` pour garder la branche à jour.

## Commits

- Commit messages clairs et en anglais/français suivant le repo. Format recommandé :
  - `feat: ajouter une nouvelle fonctionnalité`
  - `fix: corriger un bug`
  - `chore: tâches non-fonctionnelles`
  - `docs: modifications de la documentation`
- Préfère des commits atomiques et teste localement avant push.

## Pull Requests

- Ouvre une PR décrivant le but, changements majeurs, et captures d'écran si UI.
- Lier la PR à l'issue si nécessaire.
- Ajoute des tests si possible (même unitaires simples).

## Tests & Qualité

- Exécute les scripts locaux (lint, build) avant d'ouvrir la PR.
- Respecte les conventions TypeScript/ESLint du projet.

## Code Review

- Respect mutuelle et retours constructifs. Les reviewers peuvent demander des changements ; prends-les en compte.

Merci !
